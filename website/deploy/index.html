Deployment

Run the API in Docker. Keep infrastructure control on the host.

The recommended model is explicit: Docker hosts the API runtime, while `microcloud`, `lxc`, `snap`, Docker, DNS, and reverse-proxy commands execute on the target machine over SSH or Tailscale SSH.

Recommended host-operator model

This avoids fragile privilege and bind-mount combinations for host lifecycle control. The control plane remains portable, and the actual infrastructure binaries stay exactly where operators expect them.

Container role

Serve the API and hold SSH client tooling for remote host operations.

Host role

Retain `microcloud`, `lxc`, `snap`, Docker, DNS, and reverse-proxy binaries.

Reverse proxy

Caddy sidecar publishes an auto-assigned host port instead of assuming `80` or `443` are free.

Bootstrap
cd deploy/docker
cp host-operator.env.example host-operator.env
mkdir -p ssh
chmod 700 ssh
docker compose -f docker-compose.host-operator.yml up -d --build
docker compose -f docker-compose.host-operator.yml port caddy 80

Key environment variables

Configuration that defines the runtime boundary.

REMOTE_EXEC_PREFIX

Choose `ssh` or `tailscale ssh` as the remote transport wrapper.

OPERATOR_SSH_TARGET

Targets host-level Docker, Snap, VPN, DNS, reverse-proxy, and computer-use commands.

MICROCLOUD_SSH_TARGET

Routes `microcloud` execution to the target server.

LXC_SSH_TARGET

Routes `lxc` execution, falling back to `MICROCLOUD_SSH_TARGET` when unset.

REVERSEPROXY_MODE

Choose `caddy` or `nginx` so validation and config actions use the right adapter.

MICROCLOUD_AGENT_APPROVAL

Required for mutating workflows such as `configure_multi_node` and `docker_prune_everything`.

Approval and operational safety

Mutating workflows stay blocked until the environment explicitly authorizes them. That boundary is part of the product, not an afterthought.

Approval gate
export MICROCLOUD_AGENT_APPROVAL=approved
Mutating workflows

`install_microcloud_stack`, `configure_single_node`, `configure_multi_node`, and `docker_prune_everything` are gated.

Sudo path

`PRIVILEGE_EXEC_PREFIX=sudo` is the intended route once the target host is configured for non-interactive sudo.

Proxy behavior

Caddy uses `127.0.0.1::80`, which avoids startup failures from occupied host ports.
