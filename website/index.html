Why it exists

Edge infrastructure should be programmable, not fragile.

Most edge platforms stop at dashboards. UnboxdCloud is designed to carry intent all the way into controlled execution, with workflows, policies, and the real binaries operators already trust.

What it does

Turns operators into software-defined control loops.

The runtime can inspect, plan, approve, execute, report, and stream operational state across remote hosts and edge clusters.

What it avoids

No fake abstraction layer.

It does not pretend infrastructure is safe because it is wrapped in chat. Commands stay explicit. Mutations stay gated.

Built for operators

The edge needs automation that respects reality.

Agentic operations

Use chat, API requests, or workflows to move from intent to action without losing the operator’s mental model.

Remote-first execution

Reach edge hosts over SSH or Tailscale SSH, keeping the control plane containerized while execution stays close to the host.

Approval boundaries

Mutating actions remain blocked until approvals are explicit, making agentic automation usable in production.

Multi-surface runtime

Expose the same operational core through CLI, HTTP, streaming chat, Agent Kernel, and Docker deployment.

Edge control adapters

Drive MicroCloud, LXC, Docker, Snap, VPN, DNS, reverse proxy, and operator toolchains through one policy-aware runtime.

Structured observability

Return clean JSON payloads for health, workflows, commands, and deployment status rather than opaque console noise.

Runtime model

AI-native, API-first, and still grounded in shell reality.

01

Intent enters as chat, API, or workflow.

Operators can ask questions, submit structured workflow payloads, or invoke the runtime directly from automation systems.

02

Planning stays deterministic.

Every workflow resolves into explicit command steps before anything mutates the target environment.

03

Execution stays close to the host.

The Docker control plane reaches the edge host over SSH instead of trying to masquerade as the host itself.

04

Responses come back structured.

Health, workflow results, approvals, and telemetry surface as machine-usable outputs and human-readable operator context.

Deployment

Deploy the control plane once. Operate every edge site from there.

The recommended production shape is a Docker-deployed UnboxdCloud runtime plus a lightweight Caddy edge, with remote execution into the actual infrastructure host.
