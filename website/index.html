Execution model

API first

Automate everything over a clean runtime contract.

The direct HTTP server and Agent Kernel layer expose the same operational core.

Execute a workflow
curl -X POST http://127.0.0.1:8765/run \
  -H "Content-Type: application/json" \
  -d '{
    "workflow": "assess_health",
    "context": {
      "environment": "lab",
      "inventory": "ansible/inventories/lab/hosts.ini",
      "terraform_dir": "terraform/environments/lab"
    }
  }'

Deployment paths

Run anywhere. Built for real infrastructure.

Choose the runtime. The agent keeps the execution semantics.

Docker

Publish and run a hardened multi-stage container image with health checks and a non-root runtime user.

docker run --rm -p 8000:8000 \
  canonical-microcloud-agent:test

systemd

Install the provided unit and environment file for a managed Linux service.

systemctl enable --now \
  canonical-microcloud-agent-kernel

Agent Kernel

Expose `/api/v1/agents`, `/api/v1/chat`, and custom runtime routes under `/custom/*`.

  • Workflow execution bridge
  • Streaming chat
  • Plugin system compatibility
  • Long-running API runtime
