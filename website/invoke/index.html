API Reference

Invoke

Exercise the runtime API from a real client page.

This page builds request payloads for `health`, `plan`, `run`, and `chat`, then sends them to either direct HTTP mode or Agent Kernel custom routes. It handles loading state, request preview, and visible success or failure output.

Idle

Browser calls require the runtime to be reachable from this page. If the target server blocks cross-origin requests, serve this site behind the same host or add the required CORS policy on your API runtime.

  • `health` sends a GET request and ignores workflow fields.
  • `plan` and `run` send the shared workflow payload shape.
  • `chat` only sends the `message` field.

Endpoint map

What this page targets

Health

`GET /health` or `GET /custom/health` for approval state, resolved config, tool availability, and channels.

Plan

`POST /plan` or `POST /custom/plan` with the workflow context payload.

Run

`POST /run` or `POST /custom/run`, which will still fail when mutation approval is missing.
